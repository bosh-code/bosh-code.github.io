HTML vs CSS

HTML & CSS are two vital web technologies, that are used in combination, however achieve different thins. An analogy of the difference between the two would be:

HTML is like a skeleton you have raised from the dead. The bones of the skeleton are all the HTML elements you add like:

  • The head & header are the "brain" and skull
  • Then the body is the torso the entire body or "content"
    • Arms can be lists and images are the legs etc...
  • Then the footer for the feet

After assembling the skeleton back to life, we use CSS to give it features and clothes. With CSS we can choose how big or small each body part should be, the colour of it's clothes & shoes and everything else. CSS is loaded after the HTML, then applied, almost like dressing up the skeleton after raising it, except very quickly.

Control flow - If & Loops

An example control flow graph about making press coffee

In this example control flow graph we can multiple examples of control flow logic. For example, if there is not enough coffee, we grind more, else we exit the statement.

If there is no code in the else body, we can omit the keyword and body.

We can write this in pseudocode as:

if (coffeeAmount < amountRequired) {
 grind more
} else { 
 continue
}

Basically this code block says if the coffeeAmount condition is true, then grind more. So if the amount of ground coffee is less than the amount required, then grind more.

If we look further into the "grind more" block, we can see this can be written as a while loop:

if (coffeeAmount < amountRequired) {
 while (coffeeAmount < amountRequired) {
  grindMore();
  ++coffeeAmount;
 }
}

This while loop will first check the condition, contained in the (round brackets). We say while this condition is true, it will execute the code in the body of the loop repeatedly until the condition is false. Loops that have an always true condition will repeat infinitely.

For Loops

An example for loop:

for (let i = count; i > 0; --i) {
 let noun = i === 1 ? "bottle" : "bottles";
 console.log(i + " " + noun + " of beer on the wall,");
 console.log(i + " " + noun + " of beer,");
 noun = i - 1 === 1 ? "bottle" : "bottles";
 console.log("Take one down, pass it around, " + (i - 1) + " " + noun + " of beer on the wall.\n")
}

This for loop takes in a parameter count and prints the "99 bottles of beer on the wall" song, descending to 0 from the original count. First, it checks the condition for (let i = count; i > 0; --i) and creates the variable i with the value of count. Next it checks if i is greater ( > operator ) than 0 and declares i is to become i - 1 after each successful run through the loop.

The the body of the loop just runs through printing out the song, changing the bottles noun based on whether it's the last or second to last loop through. An example of the output of the code looks like this:

/**
2 bottles of beer on the wall,
2 bottles of beer,
Take one down, pass it around, 1 bottle of beer on the wall.
**/

The HTML DOM & Interactions

"The W3C Document Object Model (DOM) is a platform and language-neutral interface that allows programs and scripts to dynamically access and update the content, structure, and style of a document."

The HTML DOM or "Document Object Model" is a standard for how to get, change, add, or delete HTML elements. We can use Javascript or other programming languages to access the methods and properties of the DOM. For example, changing the text in an html <p> or paragraph element, using the HTML DOM requires using the .getElementById() method to access the .innerHTML property. We can write this in Javascript as: 

<script>
document.getElementById("foo").innerHTML = "Hello World";
</script>

The HTML DOM has numerous methods and properties for interacting with elements. You can see large list, as well as explanations at this link.

Accessing data from Arrays vs Objects

Objects

An object's properties consist of a key and value pair. An example of this basic syntax is:

var object = {
 key: 'value'
};

An object's properties can be accessed, added, removed or mutated by accessing them through Javascript's dot or brackets notation:

var person = {
 name: 'Ryan',
 age: '25'
};
person.name = "Ryan Bosher"; /** dot notation mutation **/
var tempAge = person['age']; /** bracket notation accessing **/

Arrays

Arrays are used to store multiple values in one variable. Instead of being key-value paired like objects, Arrays use indexing, beginning at 0. We can assign some values to an array like this:

var strings = ['foo', 'bar'];

To access a variable from the array, we can access it through brackets notations:

strings[0]; // returns 'foo'

Arrays also have built-in methods for adding, removing and shifting elements.

/** push() - Adds item(s) to the end of an array **/
strings.push('far');
/** pop() - Removes the last item of an array **/
strings.pop();
/** shift() - Removes the first item of an array **/
strings.shift();
/** unshift() - Adds item(s) to the front of an array **/
strings.unshift('boo');

Functions & Why

A function is just a block of Javascript code designed to perform a specific task. They are defined with the function keyword. A function is executed when called or invoked. An example function is as follows:

function example(p1, p2) {
 return p1 * p2;
} // This function returns the product of p1 * p2
  • function is the keyword to declare a function.
  • example is the name of the function and what we use to call it.
  • (p1, p2) are the parameters of the function, or what is passed in.
  • return is the keyword used to return a product from the function to the calling code.

Invoking this function and the result can be seen here:

function multiplyExample(p1, p2) {
 return p1 * p2;
}

var x = multiplyExample(4, 3); // x will hold 12

But why?

Instead of making the function above, we can just write 4 * 3 in it's place. But what if instead of 4 * 3 we were converting a temperature from Celsius to Fahrenheit, then we needed to do it 10 times. We could just write out var x = (5 / 9) * (x - 32) 10 times throughout our code, however this would become unwieldy very quickly.

With functions we can declare once and use it forever. We can declare a function that does our temperature conversion and just call it whenever we need to. Our code will be simpler and smaller.
