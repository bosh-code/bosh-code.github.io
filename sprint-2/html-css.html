Margin, Border & Padding in CSS

April 16, 2021 by Ryan This page is best viewed in Chrome using the Dev Tools.

This blog post explores the difference between the margin, border and padding properties of elements in CSS as well as the Box Model.

Margin, Border & Padding are all CSS properties, that change how elements are positioned and interact with others on a webpage. The three properties behave in similar ways and are often confused, however they function in distinct ways. Margin, Border & Padding combined are referred to as the "CSS Box Model", or Box model for short.

Margins clear an area outside the border and are always transparent. Margins can be thought of as the space between the border of one element and the next.

Four margins need to be declared if they are not to collapse onto each other:

  • margin-top: x;
  • margin-right: x;
  • margin-bottom: x;
  • margin-left: x;

Margin properties are usually defined in a clockwise order, starting from the top and moving around the element.
Margins can also be defined with shorthand. This condenses the 4 margin properties onto a single line, which is written as:

  • margin: x x x x;

The 4 values following the margin follow the clockwise order of margins.

Borders are properties that describe the usually visual borders of an element. Borders have 3 key components, Colour, Style and Width, as well as several other optional values.

Borders are similar and often confused with outlines, however they differ in a few key ways:

  • Outlines do not take up any space. They are drawn outside of an element's content.
  • Outlines also do not have to be rectangular, although they usually are.

The border CSS property is actually shorthand for the following CSS properties:

  • border-color: x;
  • border-style: x;
  • border-width: x;

Borders add a stroke to an element, before the margins start.
Borders can be transparent, dotted, thin, thick or any combination.

Borders can be written with shorthand aswell:

  • border: x style color;

Padding is similar to margin, however padding is located inside the border of an element. To create padding, either the surrounding element is grown or the content inside is shrunk.

There are four main padding properties and values, however any can be applied:

  • padding-top: x;
  • padding-right: x;
  • padding-bottom: x;
  • padding-left: x;

Padding properties are usually defined in a clockwise order, starting from the top and moving around the element.
Padding can also be defined with shorthand. This condenses up to 4 padding properties onto a single line, which is written as:

  • padding: x; equal padding on all side
  • padding: x x x x; separate padding values for all sides

The (up to) 4 values following the padding follow the clockwise order of padding.

The CSS box model is a way to bring margins, borders and padding together. It provides a simple way to think and visualise how the 3 properties are going to appear and interact with each other.

The CSS box model is more of a tool for visualising and teaching, rather than a rule, however it pays to be familiar with it.
